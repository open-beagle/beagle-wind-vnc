# Web 界面回车键闪烁问题修复说明

## 修复日期

2026-02-18

## 问题描述

在 Web 界面中按下回车键（Enter）时，页面会出现短暂的闪烁或冻结现象，表现为：

1. 视频流短暂中断（黑屏或冻结画面）
2. 音频流短暂中断
3. 页面可能显示"连接中"或"重新连接"提示
4. 持续时间约 0.5-2 秒

## 根本原因

前端代码中的全局键盘事件监听器没有状态检查：

```javascript
// 修复前的代码
handleKeyDown(event) {
  if (event.key === "Enter") {
    this.playStream();  // 无条件重启视频流
  }
}
```

**问题**：
- 无论视频流是否已经在播放，按回车都会触发 `playStream()`
- 导致正在播放的流被重新启动
- 造成短暂的中断和重连

## 修复内容

### 修改的文件

- `addons/gstreamer-web/src/app.js`

### 修改前

```javascript
handleKeyDown(event) {
  // 检查是否按下 Enter 键
  if (event.key === "Enter") {
    console.log(`handleKeyDown: key Down: ${event.key}`);
    this.playStream();
  }
}
```

### 修改后

```javascript
handleKeyDown(event) {
  // 只在显示"开启"按钮时响应回车键
  if (event.key === "Enter" && this.showStart === true && !event.repeat) {
    console.log(`handleKeyDown: Starting stream`);
    this.playStream();
  }
}
```

### 关键改动

1. **添加状态检查**：`this.showStart === true`
   - `showStart` 为 `true` 时，表示显示"开启"按钮（流未启动）
   - `showStart` 为 `false` 时，表示按钮已隐藏（流已启动）

2. **防止重复触发**：`!event.repeat`
   - 忽略长按回车键产生的重复事件
   - 避免多次触发

## 实现思路

原本有一个"开启"按钮，点击后会调用 `playStream()` 启动视频流，启动后按钮会隐藏（`showStart = false`）。

修复方案：将回车键与"开启"按钮绑定
- 按钮显示时（流未启动）：按回车键 = 点击"开启"按钮
- 按钮隐藏时（流已启动）：按回车键不触发任何操作

## 效果

### 修复前

- ❌ 流未启动时：按回车键启动流（正常）
- ❌ 流已启动时：按回车键重启流（导致闪烁）

### 修复后

- ✅ 流未启动时：按回车键启动流（正常）
- ✅ 流已启动时：按回车键不触发任何操作（正常）
