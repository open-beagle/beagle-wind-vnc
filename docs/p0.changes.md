# P0 实施变更记录

本文档记录 P0 任务完成后的具体变更内容。

## 1. 移除 fakeroot

**状态**: 🟢 已完成

**变更内容**:

### 修改的文件

1. `scripts/base/base-system-setup.sh`
2. `.beagle/nvidia-egl-base.Dockerfile`
3. `.beagle/nvidia-egl-desktop.Dockerfile`
4. `.beagle/nvidia-egl-steam.Dockerfile`
5. `.beagle/nvidia-egl-lutris.Dockerfile`
6. `nvidia/egl/Dockerfile`
7. `nvidia/egl/raw/Dockerfile`
8. `nvidia/glx/Dockerfile`

### 具体变更

**移除 fakeroot 安装**:

- 从所有 Dockerfile 和脚本中移除 `fakeroot` 包的安装

**恢复标准 sudo**:

- 移除 `mv -f /usr/bin/sudo /usr/bin/sudo-root`
- 移除 `ln -snf /usr/bin/fakeroot /usr/bin/sudo`
- ubuntu 用户现在使用标准的 sudo 命令

**修改 SHELL 指令**:

- 将所有 `SHELL ["/usr/bin/fakeroot", "--", "/bin/sh", "-c"]` 改为 `SHELL ["/bin/sh", "-c"]`

**预期效果**:

- sudo 命令恢复正常功能
- 需要 root 权限的操作可以正常执行
- 业务功能不再受 fakeroot 限制

**注意事项**:

- 容器需要适当的权限配置才能使用 sudo
- 在 Kubernetes 中可能需要调整 SecurityContext
- 建议测试所有需要 sudo 的功能

## 2. 升级 Lutris 与 VSCode

**状态**: 🟢 已完成

**变更内容**:

### 修改的文件

1. `.beagle/nvidia-egl-desktop.Dockerfile`
2. `.beagle/nvidia-egl-lutris.Dockerfile`

### 具体变更

**VSCode 自动升级**:

- 修改下载链接为 VSCode 官方 latest API
- 从 `https://code.visualstudio.com/sha/download?build=stable&os=linux-deb-x64`
- 改为 `https://update.code.visualstudio.com/latest/linux-deb-x64/stable`
- 构建时自动获取最新稳定版

**Lutris 自动升级**:

- 使用 GitHub API 自动获取最新版本号
- 从固定版本 v0.5.20 改为动态获取
- 使用 `curl -fsSL "https://api.github.com/repos/lutris/lutris/releases/latest" | jq -r '.tag_name'`
- 构建时自动下载最新版本

**预期效果**:

- 每次构建镜像时自动获取最新版本
- 不需要手动更新版本号
- 始终使用最新稳定版本

**注意事项**:

- 构建时需要网络访问 GitHub API 和 VSCode 服务器
- GitHub API 有速率限制（未认证：60次/小时）
- 建议在 CI/CD 中使用 GitHub Token 提高速率限制

## 3. 性能优化

**状态**: 🟢 已完成

**变更内容**:

### 修改的文件

1. `k8s/proxy/desktop-with-xray.yaml`
2. `k8s/transparent/desktop-transparent-proxy.yaml`
3. `docs/enable-settings-panel.md`

### 具体变更

**性能参数优化**:

- `SELKIES_VIDEO_BITRATE`: 4000 → 25000 (提高视频码率，改善画质)
- `SELKIES_FRAMERATE`: 30 → 60 (提高帧率，消除撕裂感)
- `SELKIES_ENABLE_RESIZE`: true → false (禁用动态分辨率，稳定性能)

**预期效果**:

- 固定 1920x1080 分辨率后，GPU 负载降低，帧率达到 60fps
- 提高码率后，画质改善，压缩伪影减少
- 帧率达到 60fps 后，纵向撕裂问题解决

**测试建议**:

- 重新部署 k8s 应用
- 测试帧率是否稳定在 60fps
- 测试画质是否改善（无明显伪影）
- 测试快速移动场景是否流畅
