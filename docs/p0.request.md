### 1.2 升级 Lutris 与 VSCode

**需求描述：** 升级 Lutris 和 VSCode 到最新版本，并实现自动获取最新版本

**目标：** 两个软件都能自动获取最新版本，不需要手动更新版本号

---

#### VSCode 自动获取最新版本方案

VSCode 提供了官方 API 来获取最新版本和下载链接：

**方法 1：使用 latest 链接（推荐，最简单）**

```dockerfile
# 直接下载最新版本，无需查询版本号
RUN wget -O /tmp/vscode.deb "https://update.code.visualstudio.com/latest/linux-deb-x64/stable" && \
    apt-get install -y /tmp/vscode.deb && \
    rm /tmp/vscode.deb
```

**方法 2：先查询版本号再下载**

```bash
# 获取最新版本号
VSCODE_VERSION=$(curl -s "https://update.code.visualstudio.com/api/releases/stable" | jq -r '.[0]')

# 使用版本号下载
wget "https://update.code.visualstudio.com/${VSCODE_VERSION}/linux-deb-x64/stable"
```

**其他平台：**

- Linux DEB (x64): `linux-deb-x64/stable`
- Linux DEB (ARM64): `linux-deb-arm64/stable`
- Linux RPM (x64): `linux-rpm-x64/stable`
- Linux tar.gz (x64): `linux-x64/stable`

**参考：**

- API: https://update.code.visualstudio.com/api/releases/stable
- 下载: https://update.code.visualstudio.com/latest/{platform}/{channel}

#### Lutris 自动获取最新版本方案

Lutris 在 GitHub 发布，可以使用 GitHub API 获取最新版本：

**方法 1：使用 GitHub API（推荐）**

```bash
# 获取最新版本号
LUTRIS_VERSION=$(curl -s "https://api.github.com/repos/lutris/lutris/releases/latest" | jq -r '.tag_name')

# 获取下载链接
LUTRIS_URL=$(curl -s "https://api.github.com/repos/lutris/lutris/releases/latest" | jq -r '.assets[].browser_download_url | select(endswith(".deb"))')

# 下载并安装
wget -O /tmp/lutris.deb "${LUTRIS_URL}"
apt-get install -y /tmp/lutris.deb
rm /tmp/lutris.deb
```

**方法 2：在 Dockerfile 中一步完成**

```dockerfile
# 自动获取并安装最新版本的 Lutris
RUN LUTRIS_URL=$(curl -s "https://api.github.com/repos/lutris/lutris/releases/latest" | \
    jq -r '.assets[].browser_download_url | select(endswith(".deb"))') && \
    wget -O /tmp/lutris.deb "${LUTRIS_URL}" && \
    apt-get install -y /tmp/lutris.deb && \
    rm /tmp/lutris.deb
```

**方法 3：构建 URL（如果 GitHub API 有限制）**

```bash
# GitHub releases 的 URL 格式是固定的
# https://github.com/lutris/lutris/releases/download/{tag}/lutris_{version}_all.deb

# 获取版本号
LUTRIS_VERSION=$(curl -s "https://api.github.com/repos/lutris/lutris/releases/latest" | jq -r '.tag_name')
LUTRIS_VERSION_NUM=${LUTRIS_VERSION#v}  # 去掉 v 前缀

# 构建下载链接
wget "https://github.com/lutris/lutris/releases/download/${LUTRIS_VERSION}/lutris_${LUTRIS_VERSION_NUM}_all.deb"
```

**参考：**

- GitHub API: https://api.github.com/repos/lutris/lutris/releases/latest
- Releases 页面: https://github.com/lutris/lutris/releases

### 1.3 性能分析 - Web 显示效果与性能问题

**问题描述：**

1. 显示质量不好，有压缩伪影
2. 跑动时有纵向撕裂感

**当前状态：**

- 分辨率：2560x1312（动态调整）
- 帧率：45 fps（配置 60 fps）
- 视频码率：20 Mbps（配置 4 Mbps）
- 编码器：nvh264enc（硬件编码已生效）
- 视频延迟：55 ms
- 音频延迟：263 ms

**问题分析：**

1. 帧率未达标（45fps vs 60fps）- 导致撕裂
2. 分辨率过高（2560x1312）- GPU 性能瓶颈
3. 码率偏低（20 Mbps 对于 2560x1312 不够）- 导致压缩伪影
4. 音频延迟过高（263ms）

**解决方案：**

修改启动命令中的配置参数：

```bash
# 1. 禁用动态分辨率，固定 1920x1080
-e SELKIES_ENABLE_RESIZE=false

# 2. 提高视频码率到 25-30 Mbps
-e SELKIES_VIDEO_BITRATE=25000

# 3. 如果需要动态分辨率，限制最大分辨率
-e SELKIES_ENABLE_RESIZE=true
-e DISPLAY_SIZEW=1920
-e DISPLAY_SIZEH=1080
```

预期效果：

- 固定 1920x1080 分辨率后，GPU 应该能达到 60fps
- 提高码率后，画质改善，压缩伪影减少
- 帧率达到 60fps 后，撕裂问题解决
