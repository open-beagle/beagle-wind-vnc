# P1 前端交互设计

## 当前 UI 结构

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                    视频流 (全屏黑底)                       │
│                                                          │
│                                                          │
│                  ┌──────────┐                             │
│                  │ loading  │  ← 连接中/失败/开启按钮     │
│                  └──────────┘                             │
│                                                     [FAB]│ ← 右侧半隐藏圆按钮
│                                                          │
└──────────────────────────────────────────────────────────┘
                                                      ┌────────────┐
                                                      │ 右侧抽屉    │

                                                      │            │
                                                      │ 性能仪表盘  │
                                                      │ 工具栏按钮  │

```

当前问题：右侧抽屉是一个长列表，所有功能平铺，没有层级。P1 新增的空闲超时和会话冲突 UI 需要找到合适的位置，不能让抽屉更臃肿。

---

## 设计原则

1. P1 新增的 UI 不进抽屉 — 空闲警告和会话冲突是全局性的紧急状态，用模态对话框（v-dialog）覆盖在视频流上方
2. 抽屉保持现状 — 不重构，只在需要时微调
3. 状态提示轻量化 — 用 snackbar（底部 toast）做非阻断性通知
4. 所有新 UI 都用现有的 Vuetify 1.5 组件，不引入新依赖

## 1. 空闲超时交互流程

### 1.1 状态机

```
正常使用 ──(4分钟无操作)──→ 警告倒计时 ──(60秒无操作)──→ 已超时断连
   ↑                          │                            │
   └──(任意输入操作)───────────┘                            │
   ↑                                                       │
   └──────────(点击"重新连接")──────────────────────────────┘
```

### 1.2 警告阶段 — 居中半透明对话框

触发条件：4 分钟无键盘/鼠标/手柄输入

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│              ┌─────────────────────────┐                 │
│              │                         │                 │
│              │    ⏱ 您似乎已离开       │                 │
│              │                         │                 │
│              │  如无操作，将在 47 秒后  │                 │
│              │  自动断开连接           │                 │
│              │                         │                 │
│              │     [ 我还在 ]          │                 │
│              │                         │                 │
│              └─────────────────────────┘                 │
│                                                          │
│                    (视频流仍在播放，半透明遮罩)            │
└──────────────────────────────────────────────────────────┘
```

设计要点：

- 半透明黑色遮罩（rgba(0,0,0,0.7)），视频流仍可见
- 倒计时数字每秒更新，用醒目颜色（橙色→红色渐变）
- "我还在"按钮点击后立即关闭，重置计时器
- 任何键盘/鼠标操作也会自动关闭（不需要点按钮）
- 对话框不可关闭（persistent），必须有操作才能消除

### 1.3 已超时阶段 — 断连提示页

触发条件：警告倒计时归零

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                                                          │
│              ┌─────────────────────────┐                 │
│              │                         │                 │
│              │    连接已断开           │                 │
│              │                         │                 │
│              │  因长时间无操作，        │                 │
│              │  桌面连接已自动断开。    │                 │
│              │                         │                 │
│              │    [ 重新连接 ]         │                 │
│              │                         │                 │
│              └─────────────────────────┘                 │
│                                                          │
│                    (黑屏，视频流已停止)                    │
└──────────────────────────────────────────────────────────┘
```

设计要点：

- 全黑背景（视频流已断开）
- "重新连接"按钮触发 `location.reload()`（最简单可靠）
- 不自动重连 — 空闲超时的目的就是释放资源

## 2. 会话冲突交互流程

### 2.1 新用户视角 — 桌面被占用

触发条件：连接时后端返回 PEER_BUSY

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                                                          │
│              ┌─────────────────────────┐                 │
│              │                         │                 │
│              │    ⚠ 桌面已被占用       │                 │
│              │                         │                 │
│              │  当前有其他用户正在      │                 │
│              │  使用此桌面。           │                 │
│              │                         │                 │
│              │  [ 强制接管 ]  [ 取消 ] │                 │
│              │                         │                 │
│              └─────────────────────────┘                 │
│                                                          │
│                    (黑屏，loading 状态)                   │
└──────────────────────────────────────────────────────────┘
```

设计要点：

- 出现在 loading 阶段（连接还没建立，没有视频流）
- "强制接管"按钮用红色/警告色，表示这是破坏性操作
- "取消"按钮关闭对话框，停留在 loading 页面（或跳转回主页）
- 点击"强制接管"后显示短暂的 loading 状态，然后正常进入桌面

### 2.2 被踢用户视角 — 被接管通知

触发条件：后端发送 KICKED 消息

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│              ┌─────────────────────────┐                 │
│              │                         │                 │
│              │    桌面已被接管         │                 │
│              │                         │                 │
│              │  其他用户已接管此桌面，  │                 │
│              │  您的连接已断开。       │                 │
│              │                         │                 │
│              │    [ 重新连接 ]         │                 │
│              │                         │                 │
│              └─────────────────────────┘                 │
│                                                          │
│                    (黑屏，视频流已停止)                    │
└──────────────────────────────────────────────────────────┘
```

设计要点：

- 和"已超时"页面结构一致，只是文案不同
- "重新连接"会重新走一遍连接流程（如果对方还在，又会看到"桌面被占用"）

## 3. 组件实现规划

### 3.1 新增 Vue data 属性

```javascript
// app.js data() 中新增
idleState: 'active',        // 'active' | 'warning' | 'disconnected'
idleCountdown: 60,           // 警告倒计时秒数
peerBusy: false,             // 是否显示"桌面被占用"对话框
kicked: false,               // 是否被踢
disconnectReason: '',        // 断连原因: 'idle' | 'kicked' | ''
```

### 3.2 新增 HTML 模板（v-dialog）

三个对话框都用 `v-dialog`，放在 `</v-navigation-drawer>` 之后、`<div id="video_container">` 之前：

```
1. idle-warning-dialog    — 空闲警告（persistent, 半透明遮罩）
2. disconnected-dialog    — 断连提示（空闲超时 / 被踢，根据 disconnectReason 切换文案）
3. peer-busy-dialog       — 桌面被占用（新用户看到）
```

### 3.3 新增 CSS

```css
/* 空闲警告的倒计时数字 */
.idle-countdown {
  font-size: 48px;
  font-weight: bold;
  color: #ff9800; /* 橙色 */
}
.idle-countdown.urgent {
  color: #f44336; /* 红色，最后10秒 */
}

/* 对话框统一样式 */
.session-dialog {
  text-align: center;
  padding: 32px;
}
.session-dialog .headline {
  margin-bottom: 16px;
}
```

### 3.4 文件改动清单

| 文件            | 改动                                               |
| --------------- | -------------------------------------------------- |
| `input.js`      | 新增 `lastActivityTime` 属性，在所有输入方法中更新 |
| `app.js`        | 新增 idle 检测定时器、对话框状态管理、断连处理     |
| `signalling.js` | 新增 PEER_BUSY / KICKED / FORCE_TAKEOVER 消息处理  |
| `index.html`    | 新增三个 v-dialog 模板                             |

## 4. 交互时序图

### 4.1 空闲超时

```
用户操作        前端 Input       前端 App          信令服务器
  │                │                │                  │
  │  鼠标/键盘     │                │                  │
  ├───────────────→│ 更新           │                  │
  │                │ lastActivity   │                  │
  │                │                │                  │
  │  (4分钟无操作)  │                │                  │
  │                │                │ 检测到超时        │
  │                │                │ idleState=warning │
  │                │                │ 显示警告对话框     │
  │                │                │ 开始60秒倒计时    │
  │                │                │                  │
  │  (方式A: 点击"我还在")          │                  │
  ├────────────────────────────────→│ idleState=active │
  │                │                │ 关闭对话框        │
  │                │                │ 重置计时器        │
  │                │                │                  │
  │  (方式B: 60秒无操作)            │                  │
  │                │                │ idleState=        │

  │                │                │ 断开信令连接 ────→│ 清理 session
  │                │                │ 显示断连页面      │
```

## 5. 视觉参考

所有对话框使用 Vuetify 1.5 的 `v-dialog` + `v-card`，保持和现有 UI 一致的 Material Design 风格。

配色方案：

- 空闲警告：橙色主题（warning）
- 被踢/断连：灰色主题（neutral）
- 桌面被占用：蓝色主题（info），"强制接管"按钮用红色（error）

图标（Material Icons，已有字体）：

- 空闲警告：`timer` 或 `hourglass_empty`
- 断连：`link_off`
- 桌面被占用：`desktop_access_disabled`
- 被踢：`swap_horiz`
